问题的原因在于 HTML 中使用了 Tailwind 的 `flex` 类，它覆盖了 `.page-view` CSS 类中的 `display: none` 属性。这就导致两个 Tab 实际上一直都处于渲染状态（各自占据约 50% 的屏幕高度），其中未激活的 Tab 虽然不可见（`visibility: hidden`），但仍然占据了布局空间，从而产生了大量的留白。

**修复方案：**

1. 修改 `prototype.html` 文件。
2. 找到 `view-home` 和 `view-petroom` 对应的 `div` 元素。
3. 从它们的 `class` 属性中移除 `flex` 和 `flex-col` 类。

   * `display: flex` 和 `flex-direction: column` 属性已经由 `.page-view.active` CSS 规则正确处理了。

   * 移除 HTML 中的 `flex` 类后，未激活的 Tab 将能正确应用 `.page-view { display: none }`，不再占据空间。

**代码变更详情：**

* **view-home**: 将 `class="page-view active bg-[#fffaf0] p-6 h-full min-h-0 flex flex-col overflow-hidden"` 修改为 `class="page-view active bg-[#fffaf0] p-6 h-full min-h-0 overflow-hidden"`。

* **view-petroom**: 将 `class="page-view relative flex flex-col p-0 h-full min-h-0 overflow-hidden"` 修改为 `class="page-view relative p-0 h-full min-h-0 overflow-hidden"`。
